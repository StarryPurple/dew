# 惰性求值

Dew **默认惰性求值**。编译器自动决定哪些表达式需要立即求值，哪些可以挂起。

## 隐式惰性求值

Dew 中普通代码永远不需要写 `lazy` 或 `force`。编译器自动插入这些操作：

```dew
def expensive = heavy_computation();  // 被挂起 —— 此处不求值
expensive + 1                         // 被强制 —— 需要时计算
```

### 为什么惰性

- **效率**：未使用的计算永远不求值
- **可组合性**：无限数据结构成为可能（惰性尾部）
- **递归**：为递归绑定提供自引用 thunk

## 严格 IO 规则

函数**当且仅当有副作用时严格求值**：

```dew
def greeting = Stdout("hello")     // 严格 —— 在定义时求值
def x = Stdin(0)                   // 严格 —— 在定义时读取 stdin
def f = fn() { Stdout(1) }        // 严格 —— 函数体中有 IO
def g = fn(x: Int) -> Int { x }   // 惰性 —— 纯函数，无 IO
```

编译器在定义的表达式树中检测 `Stdin`/`Stdout` 调用。若找到，发出 `StrictDef` IR 节点（立即求值）而非惰性 thunk 链。后续 FFI 函数将加入此检测。

纯定义保持惰性（thunk）。`force`/`!` 操作符在需要时提供手动严格性。

## 严格性分析

编译器对 AST 执行一次分析，将每个表达式上下文分类为**严格**或**惰性**：

| 表达式 | 上下文 | 原因 |
|--------|--------|------|
| **字面量** | 环境 | 已是 WHNF |
| **函数体** | **惰性** | 仅产生闭包值 |
| **应用** | **严格** | 需要函数值来调用；参数惰性 |
| **Let 绑定** | **惰性** 用于值 | 可延迟到使用时 |
| **If 条件** | **严格** | 必须确定分支 |
| **Force** | **严格** | 显式需求 |
| **Return** | **严格** | 必须计算才能退出 |

## Thunk Graph IR

惰性求值通过 **Thunk Graph IR** 实现——一种扁平的、基于标签的表示，包含六个基本操作：

| 节点 | 用途 |
|------|------|
| `ThunkAlloc` | 在堆上分配 thunk 单元（初始状态：Suspended） |
| `ThunkDef` | 定义 thunk 的计算体 |
| `Force` | 要求求值一个 thunk |
| `Update` | 将结果记忆化回 thunk |
| `Blackhole` | 将 thunk 标记为 Evaluating（用于循环检测） |
| `Def` | 将顶层名称绑定到 thunk |

### 三状态 FSM

```
                  Force
Suspended ─────────────────► Evaluating
    ▲                            │
    │                            │ 求值完成
    │         Update             ▼
    └─────────────────────── Evaluated
                                │
                          Force │ 命中
                                ▼
                         （返回缓存值）
```
