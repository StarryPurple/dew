# 输入/输出

Dew 通过两个内置函数和管道操作符提供极简的 I/O 系统。所有 I/O 是有副作用的，由求值器在运行时处理。

## 内置 I/O 函数

| 内置函数 | 描述 |
|----------|------|
| `Stdin` | 从标准输入读取值 |
| `Stdout` | 将值写入标准输出 |

它们不是普通函数——求值器将它们识别为 `Value::Builtin` 并特殊派发。

## 读取输入：`Stdin`


此程序：
1. 声明 `a = 0` 和 `b = 0` 作为绑定
2. 管道 `a -> Stdin` —— 从 stdin 读取整数并绑定到 `a`
3. 管道 `b -> Stdin` —— 读取另一个整数并绑定到 `b`
4. 管道 `a + b -> Stdout` —— 打印和
5. 返回 `0` 作为退出码

## 输出：`Stdout`

```dew
42 -> Stdout;            // 打印：42
a + b -> Stdout;          // 打印：a 和 b 的和
"hello" -> Stdout;        // 打印：['h', 'e', 'l', 'l', 'o']
```

## I/O 和惰性求值

I/O 操作天然是**严格**的：读取或写入必须在程序执行的特定时刻发生。由于严格性分析将函数应用分类为严格，通过 `-> Stdin` / `-> Stdout` 的 I/O 始终是急切求值的。

此外，根据严格 IO 规则，任何包含 I/O 的 `def` 定义本身也是严格的——整个定义立即求值，无条件地执行副作用。

## 注意事项

- **无 `print` 函数** —— 所有输出通过 `Stdout`
- **文件 I/O** 规划在第五阶段（尚未实现）
- **无 IO 类型** —— 严格 IO 规则替代了显式 IO 类型
